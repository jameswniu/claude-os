# Problem #2 (semantic versioning)

Source: (internal URL)

In software development determining which version of software you are running is important. Semantic versioning is a schema that follows a simple pattern.  
  
MAJOR.MINOR.PATCH  
  
Given a version number: 9.2.5  
  
\- major: 9  
\- minor: 2  
\- patch: 5

  

You are given a list of version numbers of a software; i.e. 10.10.4, 10.4.12, 10.3.4, 10.3.41. Write a program to sort the list and print it. All the version numbers are taken as strings. Example file: [versions.txt](https://confluence.centro.net/download/attachments/72417990/versions.txt?version=1&modificationDate=1552316475981&api=v2)
