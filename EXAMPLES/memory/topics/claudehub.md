##  What is Claude Code?

Claude Code is Anthropic's **agentic coding tool** that lives in your terminal and helps you build software faster. Unlike traditional code assistants that only suggest completions, Claude Code can:

  * **Understand your entire codebase** \- Ask questions, navigate complex projects

  * **Take action** \- Edit files, run commands, create commits

  * **Automate tasks** \- Fix bugs, write tests, resolve merge conflicts

  * **Build features** \- Describe what you want in plain English




Think of it as a collaborative coding partner that can read, understand, and modify your code while you focus on the bigger picture.

## 💡 Key Concepts

Understanding these core concepts will help you get the most out of Claude Code and customize it for your workflow.

**CLAUDE.md (Memory)** A markdown file in your project root that provides Claude with persistent context about your project. Include coding standards, architecture decisions, common commands, and project-specific information that Claude should always know.

**Opus vs. Sonnet vs. Haiku** Claude Code offers three models. **Haiku** is the fastest and lightest--use it for quick questions, simple boilerplate, or generating straightforward Helm values and K8s manifests. **Sonnet** is the workhorse for daily development: writing Rails migrations, debugging service code, refactoring, and most infrastructure work. **Opus** is best for complex reasoning: tracing bugs across your Rails app, understanding interactions between Java microservices, or architectural decisions requiring deep context. The default model is Opus. When you hit your Opus limit, Claude Code automatically falls back to Sonnet. Use `/model` to switch manually and `/status` to check remaining usage.

**Token Limits** Tokens are the units Claude uses to measure text (roughly 4 characters = 1 token). Your conversation consumes tokens for both input (what you provide) and output (Claude's responses).

**Context Window** The amount of text (code, conversation history, file contents) that Claude can "see" at once during a session. Claude Code's context window is approximately 200,000 tokens. When working on large codebases, Claude selectively loads relevant files to stay within this limit.

**Auto-Compaction** When a conversation approaches the context window limit, Claude Code automatically summarizes older parts of the conversation to free up space. This allows longer sessions without losing important context, though some details from earlier in the conversation may be condensed. For complex multi-step tasks, consider running `/compact` manually between logical milestones--this gives you more control over when context is summarized and reduces the chance of losing important details mid-task.
